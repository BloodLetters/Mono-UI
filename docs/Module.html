Modules

Modules are optional add-ons that enhance the MonoUI window with extra UI components. They are off by default and must be activated manually before creating the window.

You must declare MonoUI.module before calling MonoUI.CreateWindow() for modules to take effect.
Lua 
-- Declare which modules to activate BEFORE creating the window
MonoUI.module = {
    profile = true,
}

local window = MonoUI.CreateWindow({
    Title    = "mono ui",
    Subtitle = "premium modular library",
    Size     = UDim2.fromOffset(600, 400),
})

Module API

Profile

Renders a player profile widget at the bottom of the sidebar. Displays the local player's avatar (round headshot), display name, and username. Automatically reads Players.LocalPlayer — no extra setup needed.

All fields are optional. If omitted, the module reads the live local player data at runtime.

Options

Field Type Default Description
CustomName string player.DisplayName Override the display name shown in the profile widget.
CustomUsername string "@" .. player.Name Override the username handle (shown in muted colour below the name).
CustomAvatar string rbxthumb AvatarHeadShot A custom asset/URL to use as the avatar image instead of the auto-generated headshot thumbnail.
Height number 48 Height in pixels of the profile footer area.

Activation

Lua 
-- Enable before CreateWindow
MonoUI.module = {
    profile = true,
}

local window = MonoUI.CreateWindow({
    Title    = "mono ui",
    Subtitle = "premium modular library",
    Size     = UDim2.fromOffset(600, 400),
})

Custom Override Example

Lua 
-- Override name, username and avatar manually
MonoUI.module = {
    profile = {
        CustomName     = "BloodLetters",
        CustomUsername = "@bloodletters",
        CustomAvatar   = "rbxassetid://12345678",
        Height         = 56,
    },
}

local window = MonoUI.CreateWindow({
    Title    = "mono ui",
    Subtitle = "premium modular library",
    Size     = UDim2.fromOffset(600, 400),
})

Notes

  • The profile widget shrinks the tab list height to make room — the sidebar divider line stays continuous and never clips, even with many tabs.
  • Setting profile = true uses all defaults. Alternatively pass a table of options as the value to override specific fields.
  • The avatar is displayed as a circle (rounded) with a subtle stroke border.
