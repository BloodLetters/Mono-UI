Getting Started
Overview

MonoUI Documentation

A premium, modular Roblox UI library with dark-mode aesthetics, auto-saving config, live search, and autoexec support.

All interactive components automatically sync their values to the global config when modified. Use flag to identify each element.
Getting Started › Quick Start
Setup

Quick Start

Paste the loadstring bootstrapper into your script to initialize MonoUI.

Lua 
local MonoUI = loadstring(game:HttpGet("https://github.com/BloodLetters/mono-ui/releases/latest/download/Release.luau"))()

Overview

Features

MonoUI ships with a full suite of UI primitives and productivity features out of the box.

Live Search

Built-in search bar filters components in real-time as you type.

Auto Save & Load

Serializes all component states including Color3 and KeyCodes.

AutoExec

Queues script re-execution on server teleport automatically.

Control HUD

Draggable floating quick-toggle HUD with icon-only buttons.

Core › CreateControlHUD
Core API

CreateControlHUD

Creates a floating, draggable, compact HUD with quick toggles (perfect for keeping essential scripts visible).

Argument Type Description
icon string Lucide icon name for this HUD shortcut.
default boolean Initial active state of the shortcut button.
callback function Fires with active: boolean on toggle.
Lua 
MonoUI.CreateControlHUD({
    { icon = "swords", default = false, callback = function(v) print("Aim:", v) end },
    { icon = "eye",    default = true,  callback = function(v) print("ESP:", v) end },
})
Core › CreateWindow
Core API

CreateWindow

Creates the main window container. Must be called before creating any tabs or components.

Argument Type Description
AutoExec boolean Re-executes script on teleport via queue_on_teleport.
AutoExecUrl string URL of the raw script to download and execute for AutoExec.
AutoSave boolean Auto-saves component state changes.
ConfigName string Filename key used for config persistence.
DisplayOrder number Render layering display order index of the ScreenGui.
Icon string Lucide icon name shown in the corner.
Name string Name property of the ScreenGui instance.
Position UDim2 Initial screen position coordinates of the window.
Size UDim2 Window dimensions.
Subtitle string Subtitle text below the title.
Title string Window title displayed in the top bar.
Lua 
local window = MonoUI.CreateWindow({
    Title      = "mono ui",
    Subtitle   = "premium modular library",
    Size       = UDim2.fromOffset(600, 400),
    Icon       = "shield",
    ConfigName = "my_config",
    AutoSave   = true,
    AutoExec   = true,
})
Core › Notify
Core API

Notify

Fires a toast notification at the bottom right corner of the screen.

Argument Type Description
content string Description content of the notification.
duration number Time in seconds before the notification disappears.
icon string Lucide icon name displayed beside the text.
title string Title of the notification header.
Lua 
MonoUI.Notify({
    title    = "Success",
    content  = "Cheat successfully loaded!",
    icon     = "check-circle",
    duration = 5,
})
Core › SetWatermark
Core API

SetWatermark

Creates or updates a small top-right HUD showing custom text, client FPS, ping, and character coordinates.

Argument Type Description
anchorPoint Vector2 Anchor point of the watermark frame.
position UDim2 Custom screen position offset.
text string Custom prefix header text.
visible boolean Sets the visibility of the watermark HUD.
Lua 
MonoUI.SetWatermark({
    visible = true,
    text    = "MonoUI Premium",
})

CreateColorPicker

An advanced RGB/HSV color picker that supports hex input and saturation/brightness visual adjustments.

Argument Type Description
callback function Fires with the selected Color3 object.
default Color3 Initial Color3 value.
flag string Unique key for config saving.
text string Label for the color picker.
Lua 
tab:CreateColorPicker({
    text     = "ESP Color",
    default  = Color3.fromRGB(255, 0, 0),
    flag     = "esp_color",
    callback = function(color)
        print("Color changed:", color)
    end,
})

CreateDropdown

A selection menu for single or multi-value selections. Supports search filtering inside dropdown items.

Argument Type Description
callback function Fires with a table of selected options.
default string | table Pre-selected option(s).
flag string Unique key for config saving.
list table Array of string options to display.
multiple boolean Allows selecting multiple items concurrently.
text string Label for the dropdown dropdown.
Lua 
tab:CreateDropdown({
    text     = "Target Part",
    list     = {"Head", "Torso", "LeftArm", "RightArm"},
    default  = "Head",
    multiple = false,
    flag     = "target_part",
    callback = function(selected)
        print("Selected:", selected[1])
    end,
})

CreateInput

A single-line text input field for strings and custom configs.

Argument Type Description
callback function Fires with the updated text value.
default string Initial text value.
flag string Unique key for config saving.
placeholder string Ghost text shown when the input is empty.
text string Label of the input field.
Lua 
tab:CreateInput({
    text        = "Webhook URL",
    placeholder = "https://discord.com/api/webhooks/...",
    default     = "",
    flag        = "webhook_url",
    callback    = function(url)
        print("Webhook:", url)
    end,
})

CreateKeybind

Allows the user to bind a keybind shortcut to toggle/trigger functions. Click to rebind, press Escape to unbind.

Argument Type Description
callback function Fires with the new KeyCode when bound/triggered.
default Enum.KeyCode Initial bound KeyCode.
flag string Unique key for config saving.
text string Label for the keybind.
Lua 
tab:CreateKeybind({
    text     = "Toggle GUI",
    default  = Enum.KeyCode.RightControl,
    flag     = "gui_toggle",
    callback = function(key)
        print("Keybind triggered:", key.Name)
    end,
})

Components › CreateTargetBody
Components

CreateTargetBody

A body hitbox selector showing a visual representation of a character body. Perfect for selective aimbot hitboxes.

Argument Type Description
callback function Fires with a table of currently active body parts.
default string | table Initially selected body part(s) (e.g. 'Head' or {'Head', 'Torso'}).
disabledParts table List of parts that cannot be selected (e.g. {'LeftArm', 'RightArm'}).
flag string Unique key for config saving.
multiple boolean Allows selecting multiple body parts.
text string Label for the hitbox selector.
Lua 
tab:CreateTargetBody({
    text          = "Selective Hitboxes",
    multiple      = true,
    default       = {"Head", "Torso"},
    disabledParts = {"LeftArm", "RightArm"},
    flag          = "target_hitboxes",
    callback      = function(parts)
        print("Active hitboxes:", table.concat(parts, ", "))
    end,
})

CreateToggle

A checkbox-style switch for boolean values. State is persisted to config when a flag is provided.

Argument Type Description

flag string Unique key for config saving.

    default  = false,
    flag     = "silent_aim",
    callback = function(state)

    end,

Full Script

A complete working example using all major MonoUI features.

Lua 
local MonoUI = loadstring(game:HttpGet("https://raw.githubusercontent.com/YOUR_USERNAME/mono-ui/main/dist/mono-ui.luau"))()

MonoUI.SetWatermark({ visible = true, text = "MonoUI Premium" })

local window = MonoUI.CreateWindow({
    Title      = "mono ui",
    Subtitle   = "premium modular library",
    Size       = UDim2.fromOffset(600, 400),
    Icon       = "shield",
    ConfigName = "my_config",
    AutoSave   = true,
    AutoExec   = true,
})

-- Console tab
local consoleTab = window:CreateTab({ text = "Console", icon = "terminal" })
consoleTab:CreateSection({ text = "System Logs" })
local logger = consoleTab:CreateLogger({ text = "Output", height = 280 })

-- Combat tab
local combatTab = window:CreateTab({ text = "Combat", icon = "swords" })
combatTab:CreateSection({ text = "Aimbot" })
combatTab:CreateToggle({
    text = "Silent Aim", default = false, flag = "silent_aim",
    callback = function(v) logger:Log("INFO", "Silent Aim: " .. tostring(v)) end,
})
combatTab:CreateSlider({
    text = "FOV", min = 10, max = 200, default = 90, flag = "fov",
    callback = function(v) logger:Log("INFO", "FOV: " .. v) end,
})

logger:Log("SUCCESS", "All tabs loaded.")
